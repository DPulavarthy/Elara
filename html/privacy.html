The data we collect

5) Message Data (How we use the data, Section 7)

How we use the data

NOTE: All data collected is to make your experience with the services better.


1) Guilds

The guild name and ID is used to create a Database entry for your server, so you can use different features within the bot, such as warnings, kicks, bans, softbans and logging.

2) Channels

The channel IDs collected are used to set/configure the channels to do certain actions such as: logging, suggestions, welcome and leave channels, ignoring channels for logging purposes.

3) Users

The username and ID is used to create a Database entry for yourself for the currency, profile, afk and custom commands and for the server's warnings. If you wish to delete your profile for currency use: reset me in the server that you want it to be removed from. If you wish to delete your profile you'll have to join the support server and request your data to be removed. If you wish to delete your server's warnings you will have to contact that server's staff to remove them.

4) Roles

The role ID is used for the welcome/autojoin role, this is given once a user joins a server.

5) User's Guilds

The user's guilds is referring to the guilds that are returned when you authorized the services to access to them, This is used to make sure you're in the support server before accessing the website.

6) Paste/Haste Content.

The paste/haste content is stored in the database for 2-3 weeks until being deleted unless it's set as never delete in that case the paste/haste content will be stored indefinitely.

7) Message Data

NOTE: ALL message content, attachments is encrypted. content attachments authorID messageID message_type guildID channelID CreatedAt The message is stored in the database for 1-2 days, if not deleted sooner. Message data is used for the bot's message logging system for server owners or admins to use at their discretion.

8) Announcement/Say Security Logging

When you use the announce or say commands, the server, channel, user and message content(content is encrypted, to avoid any security concerns) gets logged to a channel only the developer(s) have access to. Due to events that's happened before with people abusing bots that have those commands, making the bot say something in a server then reporting it to Discord, I've opted to log that information and have the (ID: 123456789) userID of the person that used the command at the bottom of the embed for security purposes and to protect the bot for future use and so the bot doesn't get banned if someone does do that, I would have the logs of it and can prove it wasn't the bot posting that type of stuff.

TLDR; It's for security purposes, to protect the bot from bad actors.


How to remove your data from our services

For server data, it will be removed once the bot leaves the server. For user data, refer to the How we use the data section For user-guilds data this will be removed after 10 minutes to 1 day, or you can press the logout button to do it sooner. If it's for the non-bot services join the support server and request the data to be deleted. For my.elara.services go to your pastes on the service and press the delete button. For haste.elara.services go to the support server and request the haste to be deleted from the database, or wait 2-3 weeks for it to automatically be deleted. For message data this will be removed after 1 day, or has been deleted in the server's channel, if you wish to delete your server's messages from the database use: toggle log.messages yes Note: once the messages are removed they're gone forever. If you're a user and you want ALL of your messages deleted from our system then join the support server and DM one of the Bot Staff to request your messages to be removed from the system. (NOTE: This process may take up to 24 to 48 hours to fulfill, this does not mean you have 24 to 48 hours to appeal the deletion).

Who has access to the data collected

Only the developer(s) for the services have access to the information and is NEVER given out to third parties, unless ordered by law.

How long is the data stored for

The data is stored indefinitely unless one of the following cases occurs: The bot leaves the server The user requests a data deletion NOTE: Message data is stored for 1 day (SUBJECT TO CHANGE!)
